# Codex 리뷰 프롬프트 가이드

이 문서는 Codex에게 전송할 리뷰 프롬프트의 단일 소스이다.

## 리뷰 기준

1. **엣지 케이스 누락**: 빈 입력, null, 동시성, 대량 데이터, 타임아웃
2. **아키텍처 적합성**: 기존 패턴과의 일관성, 관심사 분리, 의존성 방향
3. **성능 이슈**: N+1 쿼리, 불필요한 전체 로드, 인덱스 누락, 메모리 누수
4. **보안 취약점**: 인증/인가, 입력 검증, 주입 공격, 민감 데이터 노출
5. **테스트 계획 충분성**: 커버리지, 엣지 케이스 테스트, 통합 테스트 유무

## Severity 기준

| Severity   | 기준                                   | 예시                             |
| ---------- | -------------------------------------- | -------------------------------- |
| **HIGH**   | 구현 시 버그/장애 확실, 아키텍처 위반  | N+1 쿼리, 인증 누락, 데이터 유실 |
| **MEDIUM** | 개선 권장이나 동작에는 문제 없음       | 에러 핸들링 부족, 테스트 미흡    |
| **LOW**    | 사소한 개선 있으나 구현 진행 가능      | 네이밍, 주석, 미미한 리팩터링    |
| **LGTM**   | 추가 수정 없이 진행 가능 (명시적 승인) | 모든 기준 충족, 이슈 없음        |

## 응답 형식

Codex는 아래 형식으로 응답해야 한다:

```
{START_MARKER}
### SEVERITY: [HIGH|MEDIUM|LOW|LGTM]

### 피드백
- [구체적 피드백 항목]
- ...

### 수정 제안
- [구체적 수정 제안]
- ...
{END_MARKER}
```

## 프롬프트 템플릿

send-to-codex.sh에서 사용하는 한 줄 프롬프트:

```
다음 plan 파일({PLAN_PATH})을 읽고 리뷰해줘. 기준: (1)엣지케이스 누락 (2)아키텍처 적합성 (3)성능 (4)보안 (5)테스트. 반드시 'SEVERITY: HIGH/MEDIUM/LOW/LGTM' 형식으로 심각도를 표기하고 구체적 수정안을 제시해. 응답 시작에 '{START_MARKER}'를, 응답 끝에 '{END_MARKER}'를 반드시 출력해.
```
