## 작업 회고 생성 워크플로우

사용자 요청: `$ARGUMENTS`

### 1. 기간 파싱

`$ARGUMENTS`를 분석하여 git log의 `--since`/`--until` 날짜로 변환한다.
빈 인자는 "오늘"로 처리한다.
macOS BSD date (`date -v`)를 사용한다.

**지원 형식:**

| 입력                | since               | until               |
| ------------------- | ------------------- | ------------------- |
| 오늘 (또는 빈 인자) | 오늘 00:00          | 내일 00:00          |
| 어제                | 어제 00:00          | 오늘 00:00          |
| 이번주              | 이번주 월요일 00:00 | 내일 00:00          |
| 지난주              | 지난주 월요일 00:00 | 이번주 월요일 00:00 |
| 이번달              | 이번달 1일 00:00    | 내일 00:00          |
| 지난달              | 지난달 1일 00:00    | 이번달 1일 00:00    |
| YYYY-MM-DD          | 해당일 00:00        | 다음날 00:00        |
| M/D                 | 올해 해당일 00:00   | 다음날 00:00        |
| M/D~M/D             | 시작일 00:00        | 종료일 다음날 00:00 |

날짜 계산 예시:

```bash
# 오늘
SINCE=$(date -v0H -v0M -v0S +%Y-%m-%d)
UNTIL=$(date -v+1d -v0H -v0M -v0S +%Y-%m-%d)

# 어제
SINCE=$(date -v-1d -v0H -v0M -v0S +%Y-%m-%d)
UNTIL=$(date -v0H -v0M -v0S +%Y-%m-%d)

# 이번주 (월요일 기준)
DOW=$(date +%u)
SINCE=$(date -v-$((DOW-1))d -v0H -v0M -v0S +%Y-%m-%d)
UNTIL=$(date -v+1d -v0H -v0M -v0S +%Y-%m-%d)

# 지난주
DOW=$(date +%u)
SINCE=$(date -v-$((DOW-1+7))d -v0H -v0M -v0S +%Y-%m-%d)
UNTIL=$(date -v-$((DOW-1))d -v0H -v0M -v0S +%Y-%m-%d)
```

출력 헤더에 사용할 표시 날짜와 task 유형도 결정한다:

- 단일 날짜: `2026-2-13(금)_daily_task`
- 기간 범위 (주간): `2026-2-10(월) ~ 2026-2-14(금)_weekly_task`
- 기간 범위 (월간): `2026-2-1(토) ~ 2026-2-28(토)_monthly_task`
- 기간 범위 (기타): `2026-2-10(월) ~ 2026-2-14(금)_daily_task`

날짜 표시 규칙:

- 월/일 앞에 0을 붙이지 않는다 (2-13, 1-5)
- 요일: 월, 화, 수, 목, 금, 토, 일

### 2. 저장소 탐색

`~/Documents/projects` 하위에서 `.git` 디렉토리를 탐색한다.

```bash
find ~/Documents/projects -maxdepth 2 -name ".git" -type d \
  ! -path "*.worktrees/*" \
  ! -path "*/node_modules/*" \
  ! -path "*/.next/*" \
  ! -path "*/dist/*"
```

각 `.git`의 부모 디렉토리가 저장소 경로이다.

### 3. 커밋 수집

작성자 이메일을 자동 감지한다:

```bash
AUTHOR_EMAIL=$(git config user.email)
```

각 저장소에서 해당 기간의 커밋을 수집한다:

```bash
git -C "{repo_path}" log --all \
  --author="{AUTHOR_EMAIL}" \
  --since="{SINCE}" --until="{UNTIL}" \
  --no-merges \
  --pretty=format:"%H|%s"
```

커밋이 0건인 저장소는 스킵한다.
모든 저장소에서 커밋이 0건이면 "해당 기간에 커밋이 없습니다."를 출력하고 종료한다.

### 4. 브랜치 그룹핑 + AI 요약

#### monorepo 커밋

각 커밋의 브랜치를 추적한다:

```bash
git -C "{repo_path}" name-rev --name-only {commit_hash}
```

브랜치명에서 버전 prefix를 추출하여 그룹명으로 사용한다.
예시 (프로젝트에 맞게 조정):

| 브랜치 패턴                  | 그룹명          |
| ---------------------------- | --------------- |
| `web-X.Y.Z` 또는 `web/...`   | web-X.Y.Z       |
| `app-X.Y.Z` 또는 `app/...`   | app-X.Y.Z       |
| `admin/...` 또는 `admin-...` | 어드민          |
| `infra/...`                  | 인프라          |
| `main`, `develop` 등         | 브랜치명 그대로 |

#### 외부 저장소

저장소 디렉토리명을 기반으로 그룹명을 매핑한다:

| 저장소명 패턴                     | 그룹명                   |
| --------------------------------- | ------------------------ |
| `*infra*`, `*aws*`, `*terraform*` | 인프라                   |
| 기타                              | 저장소 디렉토리명 그대로 |

#### AI 요약

각 그룹 내의 커밋 메시지들을 분석하여 의미 있는 작업 단위로 요약한다.

- 관련 커밋들을 하나의 작업 항목으로 묶는다
- 커밋 메시지의 prefix (feat, fix, refactor 등)를 참고하여 작업 성격을 파악한다
- 한국어로 간결하게 요약한다 (1줄, "~하다" 체 사용하지 않음, 명사형으로)
- 주간/월간도 날짜별 분리 없이 전체 기간 통합 그룹핑한다

### 5. 출력 및 클립보드 복사

#### 출력 포맷

**단일 날짜:**

```
## 2026-2-13(금)_daily_task
  - app-5.0.0
    - 프론트 요청사항 대응
  - web-1.4.0
    - 상담메인 페이지 리팩토링
  - 커넥트
    - RAG 기반의 상담 초안 작성 브레인스토밍
  - 협업/공유 사항
```

**기간 범위:**

```
## 2026-2-10(월) ~ 2026-2-14(금)_weekly_task
  - app-5.0.0
    - 기능1
    - 기능2
  - web-1.4.0
    - 작업 내용
  - 협업/공유 사항
```

#### 출력 규칙

- "협업/공유 사항"은 항상 마지막에 빈 항목으로 포함한다
- 커밋 0건이면 "해당 기간에 커밋이 없습니다."를 출력하고 종료한다
- 들여쓰기는 스페이스 2칸을 사용한다

#### 터미널 출력 + 클립보드 복사

결과를 터미널에 출력하고 `pbcopy`로 클립보드에 복사한다.

주의: `echo ... | pbcopy`를 별도 줄에 사용하면 Bash 도구의 전체 stdout이 사라지는 버그가 있다.
반드시 here-string을 사용한다:

```bash
echo "$OUTPUT" && pbcopy <<< "$OUTPUT"
```

복사 완료 후 메시지를 출력한다: "클립보드에 복사되었습니다."
