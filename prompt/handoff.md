# Handoff Skill - 컨텍스트 소진 대비 작업 인수인계

## 목적

Claude Code 세션에서 컨텍스트 윈도우가 부족해지기 전에, 현재 작업 상태를 `HANDOFF.md` 파일로 저장하여 다음 세션의 에이전트가 이 파일 하나만 읽고 hallucination 없이 작업을 이어갈 수 있게 한다.

auto-compact는 대화 히스토리를 자동으로 요약하지만, 작업 맥락(실패한 시도, 설계 결정의 이유, 남은 단계의 구체적 계획)이 손실될 수 있다. Handoff는 이를 명시적으로 보존한다.

## SKILL.md

아래를 `.claude/skills/handoff/SKILL.md`에 배치한다.

```markdown
---
name: handoff
description: 컨텍스트 소진 전에 현재 작업 상태를 HANDOFF.md로 저장하여 다음 세션이 이어받을 수 있게 한다
allowed-tools: Read, Write, Edit, Glob, Grep, Bash(git status:*), Bash(git diff:*), Bash(git log:*)
---

## Handoff 워크플로우

컨텍스트가 부족해지기 전에, 지금까지의 작업 상태를 HANDOFF.md에 기록하여
다음 세션의 에이전트가 이 파일 하나만 읽고 작업을 이어갈 수 있게 한다.

### 1. 현재 상태 수집

아래를 병렬로 실행한다:

- `git status` - 변경/미추적 파일 목록
- `git diff --stat` - 변경 파일 요약
- `git diff` - 구체적인 변경 내용 (너무 길면 `--stat`만 사용)
- `git log --oneline -10` - 최근 커밋 히스토리

### 2. 대화 컨텍스트 정리

지금까지의 대화를 회상하여 아래 항목을 정리한다.
기억이 불확실한 부분은 코드를 직접 읽어서 확인한다 (Read, Grep, Glob 사용).

**반드시 포함할 항목:**

- **원래 목표**: 사용자가 요청한 최종 목표가 무엇이었는지
- **구현 계획**: 전체 계획이 있었다면 그 전문 (단계별로)
- **완료된 작업**: 이미 끝난 단계, 커밋된 변경, 성공한 접근법
- **진행 중인 작업**: 현재 하고 있던 단계와 진행 상황
- **실패한 시도**: 시도했으나 실패한 접근법과 실패 이유 (다음 에이전트가 같은 실수를 반복하지 않도록)
- **남은 작업**: 아직 하지 않은 단계 목록
- **핵심 파일**: 작업과 관련된 주요 파일 경로 목록
- **설계 결정**: 왜 이런 방식을 선택했는지 (대안이 있었다면 대안과 기각 이유도)
- **주의사항**: 다음 에이전트가 알아야 할 함정, 제약, 의존성

### 3. HANDOFF.md 작성

프로젝트 루트에 `HANDOFF.md`를 작성한다. 아래 템플릿을 따른다.

# HANDOFF: {작업 제목 한 줄 요약}

## 상태: {진행 중 | 구현 완료 - 테스트 미작성 | 리뷰 대기 등}

- 브랜치: `{현재 브랜치}`
- 최신 커밋: `{커밋 해시}` {커밋 메시지}
- 미커밋 변경: {있음/없음} (있으면 간략히 설명)

### 4. 작성 원칙

- **자족적(self-contained)**: HANDOFF.md 하나만 읽으면 작업을 이어갈 수 있어야 한다. 대화 히스토리에 의존하지 않는다.
- **구체적**: "여러 파일을 수정했다" 대신 파일 경로와 변경 내용을 명시한다.
- **실패 기록 필수**: 실패한 시도를 빠뜨리면 다음 에이전트가 같은 실수를 반복한다.
- **추측 금지**: 기억이 불확실하면 코드를 직접 읽어서 확인한다.
- **간결하되 충분하게**: 불필요한 수식어를 빼되, 맥락은 충분히 제공한다.
```

## 핵심 설계 원칙

1. **자족성(Self-contained)**: HANDOFF.md 하나만으로 충분. 대화 히스토리 불필요.
2. **실패 기록**: 실패한 시도를 명시적으로 기록해 hallucination과 반복 실수를 방지.
3. **추측 금지**: 기억이 불확실하면 코드를 직접 읽어서 사실만 기록.
4. **구체적 파일 경로**: "여러 파일" 대신 정확한 경로와 변경 내용.
